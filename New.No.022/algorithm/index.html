알고리즘이란 어떤 문제를 해결하기 위한 일련의 절차나 방법을 공식화한 형태로
    표현한 것
    우리는 지금까지 알고리즘을 해왔다. 플로우차트(flowchart), 쉽게 풀어서 얘기할
    경우
    알고리즘이라고 얘기를 하면, 하나의 기능을 말한다.
    1. 조건 : 입력, 출력, 명확성, 유한성, 효율성
    2. 입력 : 하나 이상의 데이터를 입력받는다(매개변수)
    3. 출력 : 2개 이상의 다른 결과가 있어야한다.(return)
    4. 명확성 : 과정이 명확하고 모호가지 않은 명령어로 되어 있어야한다 (변수명은
    남이 봐도 알 수 있어야한다)
    5. 유한성(종결성) : 끝나야한다. 무한대로 실행되면 안된다.
    6. 효율성 : 실행이 가능해야 한다. 검증이 된다.
   
   
    시간 복잡도(big O 표현법)
    알고리즘이 끝날 때까지의 시간을 표기한다. 입력값은 n 이다.
    O(1) : 무조건 한번 돈다.
    O(log n) : n 지수? 만큼 돈다.
    O(n) : n번 돈다.
    O(n log n) : n의 n승 지수만큼 돈다.
    O(n^2): n 제곱만큼 돈다.
    O(2^n) : 2의 n승만큼 돈다.
    O(n!) : n!만큼 돈다. (팩토리얼) 화면에 표현 불가
